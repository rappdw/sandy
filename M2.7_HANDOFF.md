## What M2.7 is

Closes finding F2 (macOS Docker Desktop has no LAN isolation; iptables can't be
applied from macOS). The agent runs on a Docker `--internal` "sidecar" network
with no route off it except through a dual-homed `sandy-proxy` sidecar (a
golang→scratch Go binary in `proxy/`). Works identically on Linux and macOS
because it relies on `--internal` routing, not iptables. Tri-state
`SANDY_EGRESS_PROXY`: `0` off (legacy iptables on Linux), `1` permissive (block
private/LAN/metadata, allow internet — **now the default**), `2` strict
(allowlist only).

## Bugs found during live bring-up (all fixed — do NOT reintroduce)

Static tests and the topology spike could not catch these; only running it on
real Docker did. Each is a committed fix with a regression test:
- **`OS` unbound** before the network setup → defined `OS` early (`99ba6e0`).
- **Proxy booted on the `--internal` sidecar** → no default route, upstream dials
  failed (ECONNRESET). Boot on egress, attach sidecar with `--ip` (`262bdea`).
- **Networks created before the cleanup trap** → leaked on any failure, exhausted
  Docker's address pool. Create after the trap is armed (`48328e4`).
- **Transparent path sent the ClientHello twice** (`up.Write(prefix)` + splice of
  the un-consumed `bufio.Peek` buffer) → TLS "protocol version" alert. Drop the
  explicit write (`747262c`). Also fixed `extractHTTPHost` premature-truncation.
- **Headless allocated a TTY** → gemini (Ink/React) busy-looped. And **headless
  attached an interactive stdin** → `codex exec` blocked reading it. Final rule
  (`35306a8`): interactive `-it`; headless + terminal-stdin → no `-i`/`-t`;
  headless + piped-stdin → `-i`. Integration harness feeds `</dev/null`.

These headless fixes are **independent of the proxy** and fix the long-standing
GH-Actions red (agent sections timing out since ~May 6).

## Gotchas for a fresh context

- **Multiple clones, one not shared.** The dev container's `/home/claude/dev/sandy`
  shares `.git` with the macOS host (bind mount) — commits appear there
  immediately. The **Linux Spark is a separate clone** — it needs `git pull` to
  see new commits. "Still broken after a fix" usually = un-pulled clone.
- **Proxy image is ref-pinned and won't auto-rebuild** when the source at that ref
  changes (the Dockerfile content/hash is identical). Force a rebuild during dev
  with `docker rmi -f sandy-proxy`, or `sandy --rebuild`. Non-issue after merge
  (a release pins a version tag that only moves on release).
- **Don't `git worktree add` from inside the dev container** — it writes into the
  shared `.git/worktrees` and claims the branch on the host (paths don't resolve
  cross-machine). Work in the checkout directly, or use an independent `git clone`
  in the container's private `/tmp`.
- **`SANDY_DEBUG_PROXY=1`** dumps the proxy's logs at cleanup and runs a resolver
  probe (resolv.conf + `getent` + `curl remote_ip`) right after the proxy starts —
  the fastest way to bisect a proxy issue (the proxy is `scratch`, no `exec`).

## Draft squash-merge PR description (paste into the PR, then delete this section)

**Title:** M2.7: cross-platform egress proxy (closes F2)

**Body:**

Adds a Docker `--internal` egress-proxy sidecar so network isolation works
identically on macOS and Linux — closing finding F2 (macOS Docker Desktop has no
LAN isolation and iptables can't be applied from macOS).

**Design.** The agent runs on an `--internal` "sidecar" network with no route off
it except through a dual-homed `sandy-proxy` container (a `golang`→`scratch` Go
binary in `proxy/`). Tri-state `SANDY_EGRESS_PROXY`: `0` off (legacy iptables on
Linux), `1` permissive (block private/LAN/metadata, allow internet — the new
default), `2` strict (allowlist only). The proxy does DNS-redirect + transparent
SNI/Host demux (TLS never terminated) + CONNECT for git-over-SSH + a local-LLM
forward. `SANDY_ALLOW_HOSTS` (privileged) extends the allowlist.

**Default flip.** `SANDY_EGRESS_PROXY` now defaults to `1` on both platforms —
a security upgrade for Linux (closes DNS exfil + host-gateway reach) and real
isolation on macOS. iptables stays as the `=0` fallback; retiring it to a single
mechanism is a soak-data-gated POST_1.0 item (the permissive proxy only carries
:80/:443/CONNECT/DNS, so non-web public ports that work under iptables need
SANDY_ALLOW_HOSTS — watch for this in the soak).

**Validation.** Live-tested end-to-end on real macOS Docker Desktop (F2 closed:
public internet reachable, RFC1918 blocked, Claude Code runs through it) and on
Linux (integration `§13` — networks up, agent reaches the API through the proxy,
no leak, opt-out works). Unit suite + `proxy/` Go tests green.

**Not included / known-separate:** codex v0.138 auth (its env-var path no longer
feeds the Responses API) is a pre-existing agent issue, tracked in
POST_1.0_IDEAS.md — it does not involve the proxy.

Post-merge: `SANDY_PROXY_REF` is no longer needed (the proxy image builds from
`main`, which now has `proxy/`). Then run the 7-day soak with the proxy default-on.
